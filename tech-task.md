Техническое задание (ТЗ)
Тестовое задание
1. Общая информация
Цель тестового задания — разработать парсер Telegram-канала с интеграцией Telegram-бота, который автоматически обрабатывает заявки и пересылает их руководителю и менеджеру, а также отображает в боте спарсенные данные из Telegram-канала.
2. Сроки выполнения
Срок выполнения тестового задания — 2 (два) календарных дня с момента получения задания.
3. Описание задачи
Необходимо разработать систему, состоящую из:
- Парсера Telegram-канала команды;
- Telegram-бота для приема и обработки заявок;
- Механизма пересылки заявок ответственным лицам (руководитель и менеджер).

Система должна работать стабильно, быть расширяемой и готовой к дальнейшему развитию.
4. Функциональные требования
4.1. Технологический стек:
- Язык разработки: Python;
- Использование библиотек: aiogram + Telethon;
- Читаемый, структурированный код (разделение на модули);
- Наличие инструкции по запуску проекта;
- Кандидат использует свои собственные API_ID и API_HASH (Telegram).

4.2. Парсер Telegram-канала:
Telegram-канал используется как новостной канал команды.

Требования к парсеру:
- Подключение к указанному Telegram-каналу;
- Считывание новых сообщений в режиме реального времени;
- Извлечение ключевой информации из постов:
  • тип услуги / проекта;
  • описание выполненного заказа;
  • дата публикации;
- Сохранение распарсенных данных (БД или файл);
- Подготовка данных для дальнейшего использования в Telegram-боте.

4.3. Telegram-бот:
- Отображение спарсенных данных из Telegram-канала через отдельную кнопку или команду;
- Прием заявок от пользователей в формате диалога;
- Валидация вводимых данных (имя, контакт, описание задачи и т.д.);
- Формирование заявки в структурированном виде;
- Отправка заявки руководителю и менеджеру в Telegram;
- Ведение статистики заявок (день / неделя / месяц);
- Возможность изменения статуса заявки:
  • Новая
  • В работе
  • Завершена.

4.4. Уведомления:
- Отправка уведомлений при поступлении новой заявки;
- Возможность настройки списка получателей уведомлений (руководитель, менеджер).
5. Результат выполнения
На выходе ожидается:
- Исходный код проекта;
- Файл README.md с описанием установки и запуска проекта;
- Краткое описание архитектуры решения (структура проекта, взаимодействие компонентов).
